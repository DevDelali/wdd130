Some basic styling has been applied in order to illustrate the elements structure and boundaries.

Task 1: Swap Block and Inline Elements

In the HTML of this document, change this div element to a span element.
This is a <div> (block element) — it starts on a new line and stretches the full available width.
This is a <span> (inline element) — it stays in the flow and only takes the width of its content.

What happens to the layout? Does it still take up the full width?

Answer: The <div> (block) appears on its own line and fills the width; the <span> (inline) stays within the line and only as wide as its text.

Add another inline element like <em> or <strong> inside the paragraph. Observe how it flows within the line.

Answer: Inline elements (like <em> and <strong>) stay on the same line and only take as much width as their content. They don't force a line break.

Task3: Nesting Block in Inline

Try placing a <div> inside a <span> element.
Placed a Nesting Block in an Inline Block to see the outcome

What error or layout issue do you see? Why does this happen?

Answer: Placing a block-level <div> inside an inline <span> is invalid HTML. Browsers may move the block out of the inline or render unexpected spacing—so avoid nesting block elements inside inline elements.
